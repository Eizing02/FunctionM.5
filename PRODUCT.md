# PRODUCT.md

## Product Name

**Function Lab M.5**
สื่อการเรียนรู้เรื่อง “ฟังก์ชัน” สำหรับนักเรียนชั้นมัธยมศึกษาปีที่ 5

---

## Product Summary

Function Lab M.5 is a modern, hi-tech, luxury-style interactive learning web app for teaching mathematical functions to Thai Grade 11 students.

The product helps students understand functions through visual exploration instead of long explanations. Students can choose a function type, adjust parameters, see 2D and 3D visualizations, compare graph transformations, and observe how formulas affect graphs in real time.

This is not a quiz game and not a score-collecting system. It is a polished classroom learning media tool that can be shared by link and used by anyone.

---

## Product Type

**Product surface**

This is primarily an educational interactive product, not a marketing landing page.

The design should feel premium and futuristic, but the main priority is still clarity, learning, usability, and classroom readability.

---

## Target Users

### Primary Users

Thai high-school students, Mathayom 5, learning the topic of functions.

Student characteristics:

* May not be confident with abstract mathematics.
* Understand better through visuals, movement, and examples.
* Need clear feedback when changing formulas or parameters.
* Often use school computers, tablets, or smartphones.
* Need Thai language support with short, readable explanations.

### Secondary Users

Thai mathematics teachers who want to use the website as classroom media.

Teacher needs:

* Open the website quickly.
* Project it on a classroom screen.
* Explain graph behavior visually.
* Switch between function types easily.
* Use it without login, setup, or scoring system.

---

## Core Learning Goal

Students should understand that a function is a relationship where each input has exactly one output, and that changing a formula changes the graph in predictable ways.

The product should make students feel:

> “I can see what the formula is doing.”

## Main Experience

The main screen should feel like a “mathematics control room” or “function laboratory.”

A recommended layout:

* Left panel: function type selector and formula controls
* Center stage: main 2D/3D graph visualization
* Right panel: short explanation, key observation, and formula meaning
* Bottom area: transformation timeline, examples, or mini concept cards

The center graph area is the hero of the product. The interface should guide students toward manipulating functions and observing changes.

## Core Features

### 1. Function Selector

Students can choose different function types such as:

* Linear
* Quadratic
* Polynomial
* Rational
* Exponential
* Logarithmic
* Absolute value
* Piecewise
* Composite
* Inverse

The selector should feel smooth, modern, and tactile. Avoid boring dropdown-only UI if a more visual segmented selector or carousel works better.

---

### 2. Formula Panel

Students can view and adjust formulas such as:

* `y = ax + b`
* `y = ax² + bx + c`
* `y = a(x - h)² + k`
* `y = a · b^x`
* `y = log_b(x)`
* `y = |x|`
* piecewise-defined functions

Formula controls should be simple.

Use sliders, steppers, toggles, and small input fields instead of overwhelming students with too many controls at once.

---

### 3. Graph Visualization

The graph should update immediately when values change.

Graph behavior should be smooth and educational:

* Lines move when slope/intercept changes.
* Parabolas open wider/narrower when `a` changes.
* Graphs shift left/right/up/down when `h` and `k` change.
* Important points can appear with labels.
* Domain and range can be highlighted visually.
* Multiple functions can be compared.

The graph is not decoration. It is the main teaching tool.

---

### 4. 2D and 3D Learning Modes

The product should support multiple learning modes:

#### 2D Mode

Used for normal classroom explanation.

* Clear coordinate plane
* Function graph
* Points, intercepts, vertex, asymptotes where relevant
* Domain/range highlights

#### 3D Mode

Used as a visual enhancement.

The 3D mode should make the product feel modern and impressive, but it must not make the math harder to understand.

Possible 3D uses:

* Floating formula cards
* Layered graph panels
* 3D coordinate stage
* Function transformation shown as depth layers
* Multiple functions stacked as transparent planes
* Camera orbit or subtle parallax

Do not use 3D just for decoration. It should support curiosity and visual engagement.

---

### 5. Explanation Panel

Explanations must be short and meaningful.

Avoid long paragraphs.

Use this structure:

* What changed?
* Why did it change?
* What should students notice?

Example:

> เมื่อค่า `a` มากขึ้น กราฟพาราโบลาจะแคบลง
> ถ้า `a` เป็นลบ กราฟจะคว่ำลง

Use short Thai text, clear icons, and visual markers.

---

### 6. Animation and Motion

Animation is required, but it must feel premium and purposeful.

Use animation for:

* Switching function types
* Swapping panels
* Expanding/collapsing formula cards
* Moving graph layers
* Morphing graph shapes
* Highlighting changed parameters
* Showing before/after transformations
* Sliding explanation cards
* Smooth transitions between 2D and 3D mode

The interface should feel alive and pleasant to explore.

Motion should never distract from learning.

---

### 7. Swap / Window / Box Interaction

The product should include dynamic UI behavior such as:

* Swapping explanation cards
* Floating formula windows
* Resizable graph/detail panels
* Slide-over lesson panels
* Expandable concept boxes
* Compare mode with split panels
* Before/after transformation cards
* Drag-to-reorder function layers if useful

These interactions should feel smooth, premium, and intentional.

Avoid random animation. Every movement should communicate that something changed.

---

### 8. Minimal Text Philosophy

The interface should reduce unnecessary text.

Use:

* Icons with labels
* Short headings
* Formula-first explanation
* Visual examples
* Tooltips
* Progressive disclosure
* “Show more” only when needed
* Visual highlights instead of long written explanation

Do not fill the screen with textbook content.

The product should teach by showing, not by overwhelming students with reading.

## Content Voice

Language should be Thai, simple, short, and direct.

Use student-friendly wording.

Good examples:

* “ลองปรับค่า a แล้วดูความชัน”
* “กราฟเลื่อนขึ้นเมื่อค่า k เพิ่มขึ้น”
* “จุดนี้คือจุดตัดแกน y”
* “โดเมนคือค่าของ x ที่ใช้ได้”
* “เรนจ์คือค่าของ y ที่เกิดขึ้นได้”

Avoid:

* Long academic paragraphs
* Complex formal Thai
* Marketing copy
* Vague motivational text
* Buzzwords
* English-heavy explanations unless necessary for formula or UI labels

## UX Principles

### 1. Graph First

The graph should be the main focus.

Controls and explanations support the graph, not compete with it.

---

### 2. One Concept at a Time

Do not show too many formulas, sliders, panels, and explanations at once.

Reveal complexity gradually.

---

### 3. Change Must Be Visible

When students adjust a value, the screen should clearly show what changed.

Use highlight, motion, labels, color, or ghost graph comparison.

---

### 4. Minimal Text, Maximum Meaning

Use fewer words, stronger visuals, and better examples.

---

### 5. Premium but Readable

The design can be luxurious and hi-tech, but text, formulas, controls, and graphs must remain readable.

---

### 6. Classroom Friendly

The product must work well when projected on a classroom screen.

Important labels and formulas must be large enough.

---

### 7. Mobile Friendly

Students may use phones.

The mobile version should prioritize:

* Function selector
* Graph
* Main parameter controls
* Short explanation

Use stacked panels and swipeable cards.

---

## Anti-References

Avoid these patterns:

* Generic SaaS hero section
* Purple/cyan AI gradient everywhere
* Neon cyberpunk overload
* Glassmorphism on every card
* Card grid repeated across the page
* Icon tile above every heading
* Meaningless stats section
* Fake testimonials
* Marketing phrases like “supercharge learning”
* Overly cute cartoon UI
* Too many paragraphs
* Too many panels visible at once
* Plain Bootstrap-like school website
* Boring worksheet style
* UI that looks impressive but is hard to use

---

## Accessibility Requirements

The product must be accessible and readable.

Requirements:

* Thai text must be easy to read.
* Body text should generally be at least 16px.
* Important classroom text should be larger.
* Contrast must be strong enough for projection.
* Interactive controls must have clear focus states.
* Buttons and sliders must be usable on touch devices.
* Animations must respect `prefers-reduced-motion`.
* Color must not be the only way to communicate meaning.
* Graph labels must be readable.
* Controls must have accessible labels.
* Avoid tiny text inside graph areas.

---

## Responsive Requirements

### Desktop / Classroom Screen

Use a dashboard layout:

* Left control panel
* Center graph stage
* Right explanation panel
* Optional bottom concept strip

### Tablet

Use two-column or split layout:

* Graph on top or center
* Controls beside or below
* Explanation as slide-over or card

### Mobile

Use stacked and swipeable layout:

* Function selector
* Graph
* Primary controls
* Explanation card
* Details hidden behind “เพิ่มเติม”

---

## Performance Requirements

The product should feel smooth.

* Keep animations lightweight.
* Avoid heavy 3D effects on low-end devices.
* Lazy-load 3D mode if needed.
* Prefer GPU-friendly animation using transform and opacity.
* Avoid animating width, height, margin, padding, top, or left.
* Keep initial load fast.
* Provide fallback if 3D cannot run smoothly.

---

## Success Criteria

The product is successful if:

1. Students can understand the connection between formula and graph.
2. Teachers can use it without setup.
3. The interface looks premium and modern.
4. The design does not look like a generic AI-generated website.
5. Students can interact without needing long instructions.
6. The graph and formula remain readable on classroom screens.
7. Motion feels useful, smooth, and enjoyable.
8. The site works on desktop, tablet, and mobile.
9. The product feels like a real educational tool, not a template.
